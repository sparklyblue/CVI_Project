## Image Naming

Every image is named `{flight_id}_{frame_id}.jpg`, for example:

```
159_2187.jpg   →   flight 159, frame 2187
```

The corresponding label file has the exact same stem:

```
images_thermal/images/train/159_2187.jpg
labels_final/train/159_2187.txt
```

---

## Label Format

Each `.txt` file in `labels_final/` contains one line per animal in that image.
The format is standard YOLO with one extra column at the end for motion:

```
species_id  cx  cy  width  height  motion_id
```

- `species_id` — integer, see species table below
- `cx cy` — bounding box centre, normalised 0–1 relative to image size
- `width height` — bounding box size, normalised 0–1
- `motion_id` — integer, see motion table below

Example file with two animals:

```
1 0.968799 0.098975 0.062207 0.052441 0
1 0.875098 0.154492 0.078320 0.058984 1
```

Both are Red deer (class 1), one static, one moving.

| ID | Meaning |
|----|---------|

## How the Labels Were Generated

The original `labels_matched_thermal/` files had every animal as class `0` with no species information. The real labels — including species name and motion state — live in the `mots/` files, which are per-flight tracking files from the BAMBI dataset.

`build_labels.py` joins them together by looping over every image in the dataset and taking one of two paths:

**Path A — label file exists in `labels_matched_thermal/`:**
1. Reads the existing YOLO bounding boxes
2. Finds all mots detections for that flight + frame
3. Matches each YOLO box to a mots detection using bounding box IoU
4. Writes the matched species ID and motion ID into `labels_final/`

**Path B — no label file exists (most val and many train/test images):**
1. Goes directly to the mots file for that flight + frame
2. Converts the pixel-coord bounding boxes to normalised YOLO format using image dimensions
3. Writes species ID and motion ID into `labels_final/`

**Path C — no label file and no mots entry:**
Writes an empty `.txt` file (valid background frame for training).

All 72 547 boxes were matched with 0 unmatched, meaning every label has a confirmed species and motion assignment.

## How to Use the Labels for Training

### Loading a sample in Python

```python
from pathlib import Path
from PIL import Image

def load_sample(stem, split="train"):
    img_path   = Path(f"images_thermal/images/{split}/{stem}.jpg")
    label_path = Path(f"labels_final/{split}/{stem}.txt")

    image  = Image.open(img_path)
    iw, ih = image.size

    boxes = []
    with open(label_path) as f:
        for line in f:
            parts = line.strip().split()
            if len(parts) < 6:
                continue
            species_id = int(parts[0])
            cx, cy, w, h = float(parts[1]), float(parts[2]), float(parts[3]), float(parts[4])
            motion_id  = int(parts[5])

            # Convert to pixel coords if needed
            x1 = int((cx - w / 2) * iw)
            y1 = int((cy - h / 2) * ih)
            x2 = int((cx + w / 2) * iw)
            y2 = int((cy + h / 2) * ih)

            boxes.append({
                "species":   species_id,
                "motion":    motion_id,
                "bbox_norm": (cx, cy, w, h),    # normalised
                "bbox_px":   (x1, y1, x2, y2),  # pixel coords
            })

    return image, boxes
```

### Using with Ultralytics YOLO

The `data.yaml` file is already set up for Ultralytics. Point your training script at it:

```python
from ultralytics import YOLO

model = YOLO("yolov8n.pt")
model.train(data="data.yaml", epochs=50, imgsz=640)
```

> Note: Ultralytics does not natively support the extra `motion_id` column. Either train two separate heads (species + motion), strip the last column for a species-only model, or write a custom dataset class that reads both.

### Stripping motion column (species-only training)

```python
# Produces standard 5-column YOLO labels without the motion field
from pathlib import Path

for split in ["train", "val", "test"]:
    src = Path(f"labels_final/{split}")
    dst = Path(f"labels_species_only/{split}")
    dst.mkdir(parents=True, exist_ok=True)
    for f in src.glob("*.txt"):
        lines = f.read_text().splitlines()
        stripped = [" ".join(l.split()[:5]) for l in lines if l.strip()]
        (dst / f.name).write_text("\n".join(stripped))
```

## Verification

To visually check that labels are correct:

```bash
source venv/Scripts/activate   # Windows
# source venv/bin/activate     # Mac/Linux
python check_labels.py
# open http://localhost:5001
```

The checker lets you filter by species, motion state, and split, and shows images with bounding boxes and labels drawn on top. Click any image to zoom in.

## Setup & Reproducing the Labels

```bash
# 1. Create and activate virtual environment
python -m venv venv
source venv/Scripts/activate   # Windows
# source venv/bin/activate     # Mac/Linux

# 2. Install dependencies
pip install -r requirements.txt

# 3. Regenerate labels_final/ from scratch
python build_labels.py
```

This reads `mots/` and `labels_matched_thermal/` and rewrites `labels_final/`, `data.yaml`, and `label_summary.csv`.
